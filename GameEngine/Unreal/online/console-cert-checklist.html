主机过审清单：TRC / XR / Lotcheck 要实现哪些功能

把一款跨平台多人在线游戏搬上主机（PS5、Xbox Series X|S、Switch），除了画面和性能，还有一大块平台合规工作绕不过去：Sony 的 TRC（Technical Requirements Checklist）、Xbox 的 XR（Xbox Requirements）、任天堂的 Lotcheck。这篇按功能模块整理一份通用清单——具体条款各平台 DevNet 上会随固件版本更新，这里只记需要做哪些事以及踩过的坑。

Switch 上通常只需实现其中一部分：登录鉴权、商店与购买、断线处理、家长控制、术语检查、第一方 Web API 调用频率限制、从后台恢复游戏。PS5/Xbox 则基本要全套。

登录与鉴权（Login & Authentication）

把第 9 世代主机（PS5/XSX）的平台身份鉴权集成进游戏，客户端侧做完整设计与实现，并支持 SSO（单点登录）。如果游戏自有一套账号服务，还要把平台账号与自有账号做关联（account linking）。

在线状态（Presence）

把平台的 Presence 功能与游戏内/自有的在线状态同步起来，让平台好友列表里看到的状态和游戏内一致。PS5 上的方案通常对 XSX 也适用。

组队与邀请（Party & Invitation）

这是跨平台多人最复杂的一块。如果原本是用第三方库（如 Vivox）做的「纯客户端、无服务器」组队（靠 Control Message 实现），它没法直接适配主机的会话系统：

  • PS5 的 Player Session
  • XSX 的 Multiplayer Session Directory（MPSD）

需要在两个平台上各做一遍调研，给出 GDD/TDD，并把组队信息与 Player Session / MPSD 双向同步。要做真正的跨平台邀请，还要在后端起一个「groups service」之类的服务来托管组队关系——无论玩家当前在哪个平台，发起组队时都把信息同步到对应平台的会话里，才能跨平台邀请好友。

踩坑提醒：

  • 跨平台邀请的「最优解」高度依赖项目本身。如果游戏已经在 PC/Switch 上线，主机版只能做小心、克制的重构，往往会留下不少 legacy 代码还在跑。
  • 是否引入平台的专用后端（如 PS5 app server）值得评估，它可能带来更顺畅的组队/邀请体验，但成本也更高。
  • 这块功能和 gameplay、游戏设计强绑定，别照搬，只能当参考。

Xbox 官方有跨网络游玩（Cross-Network Play）的示例：

  • MMO 示例：https://developer.microsoft.com/en-us/games/xbox/docs/gdk/xr007-mmo-example
  • 多人（大厅）示例：https://developer.microsoft.com/en-us/games/xbox/docs/gdk/xr007-multiplayer-example

PS5 Activity

调研 PS5 的 Activity 功能，在后端配置 app，客户端侧完成 Activity Cards 的实现。

PS Plus 专属内容

在线多人游戏在 PS5 上通常要求 PS Plus 会员才能游玩。需要调研 PS Plus 专属内容（Exclusive Pack）的要求并实现对应的检测与解锁。

商店与购买（Store & Purchase）

实现在游戏内访问平台商店、购买虚拟货币、并实时更新游戏内货币数量。客户端侧要做：平台 API 集成、token 刷新、后端服务 API 对接、UI 适配。还要在平台 Partner Center 配置 add-on / DLC 项。

成就 / 奖杯（Trophies / Achievements）

在 PS5（Trophies）和 XSX（Achievements）的 Partner Center 配置成就，并在游戏内对接解锁逻辑。

断线处理（Network Disconnection）

枚举各种断线场景，给玩家对应的反馈提示，别让玩家卡在无响应的界面里。

家长控制（Parental Controls）

Switch 上通常需要自己设计实现家长控制；PS5/Xbox 上这块多由平台/客户提供。

杂项（Miscellaneous）

这些零碎项也都在 TRC/XR 范围内：

  1. 术语检查：修改文案和文本以符合平台术语规范（比如按钮名称、专有名词）。
  2. 第一方 Web API 调用频率检查（别超频导致被限流）。
  3. 从后台恢复游戏（resume from background）。
  4. 把 app 图标/图片打进包里。
  5. 关闭 PS5 的 PlayGo、XSX 的 Instant Play（视当前固件 TRC/XR 是否还要求）。
  6. 在 TitleConfiguration.json 和 .ini 里做对应配置。
  7. PS5 打包流水线里加 --for-submission 等提交专用参数。

跨平台多人的额外要求

跨平台联机会触发平台的额外政策，必须额外遵守。

PS5

要遵守一份 Cross-Platform Policy。这份文档在 PS5 DevNet 上搜不到，只有在 dev support 提工单申请后才会提供。设计/技术上它额外要求：

  • 设置里要有一个「仅 PlayStation」开关；
  • 账号要能与发行商账号关联；
  • 奖杯只能通过 PlayStation 上的游玩解锁；
  • PlayStation 玩家可被识别；
  • 提供 PlayStation 好友列表；
  • UGC 内容要过滤脏话（profanity）。

所有跨平台实现都必须先通过 PS5 DevNet 的 cross-platform proposal 经 Sony 批准。

Xbox

要求列在 XR-015 和 XR-045 里：游戏不得允许与用户隐私设置不符的通信和数据传输。

Xbox 用户隐私设置路径：主机 → 设置 → 账户 → 隐私和在线安全 → Xbox 隐私 → 查看详情并自定义 → 通信和多人游戏。

  • https://developer.microsoft.com/en-us/games/xbox/docs/gdk/xr015
  • https://developer.microsoft.com/en-us/games/xbox/docs/gdk/xr045

分工提醒

做主机移植/联合开发时，要分清哪些活自己能干、哪些必须找项目方（项目 owner）配合。一般需要项目方介入的工作有这些特征：

  1. 需要在平台 Partner Center 配置——可以让项目方配，或申请账号权限自己配：
    • 申请新的第一方功能（如 PS5 app server）；
    • 平台功能配置（PS5：Trophies、Game Intent；Xbox：Achievements、Session template）；
    • 项目元数据（商店图标、语言）；
    • 定价；
    • DLC 与虚拟货币。
  2. 需要后端服务对接第一方功能——主要是多人游戏：
    • 玩家登录与鉴权；
    • 游戏内组队与邀请；
    • 购买。

如果某项工作是纯客户端、不需要后端或第一方 Partner Center 支持，那就自己实现。
